# Contributing to SYNS

Thanks for helping improve Synaptic Swarm Protocol.

## Ground rules

- Keep all code, comments, and documentation in English
- Prefer small, reviewable pull requests with a clear narrative
- Open an issue before starting large design or architecture changes
- Update docs when changing behavior, workflow, or public terminology
- Do not mix unrelated changes in one pull request

## Local workflow

1. Fork or branch from `main`
2. Install dependencies with `npm install`
3. Run `npm run dev` for local development
4. Run `npm run lint`, `npm run typecheck`, and `npm run build` before opening a PR

## Pull request expectations

- Explain what changed and why
- Link the relevant issue when applicable
- Include screenshots for UI changes
- Note any follow-up work explicitly

## Commit and release guidance

- Use concise commit messages
- Keep semantic version tags for release points such as `v0.1.0`
- Update `CHANGELOG.md` when making user-visible changes that deserve release notes

## Design direction

The project visual language should stay dark, protocol-oriented, and neural-system inspired. Preserve the existing typography, grid treatment, motion restraint, and glow palette unless a deliberate brand update is part of the change.
